# Gamio UI/UX Audit — 2026-07-01

## Summary

The site works, but it is really two front-ends stitched together: a modern React/shadcn/Tailwind-v4 shell wrapped around a large ported static blob (`src/gamio-body.html` + `public/css/index.css` + `public/css/tailwind-build.css` + `public/gamio.js`). Almost every meaningful defect traces back to that seam — dark mode, design tokens, icons, a11y tooling coverage, and DOM ownership all fork at the boundary. The single biggest systemic issue is that **dark mode is visibly broken on every public page** because the blob has zero `.dark` styling and its colors are hardcoded. Everything else is smaller, but the same shape: two implementations of the same thing that only agree by accident.

No changes have been made. This is Phase 1 only.

## Systemic issues

- **The React/blob seam is the source of ~half the findings.** Findings 1, 3, 4, 5, 6, 9, 10, 11, 14 all exist only because a large chunk of the UI lives outside React's rendering, styling, and tooling. Migrating the blob to JSX is a separate project (out of scope, and correctly noted as such in AGENTS.md). The realistic mitigation is to (a) let the legacy CSS respond to `.dark` via a small override block, (b) unify the theme toggle, and (c) accept the blob for what it is until a dedicated migration is prioritized.
- **Three color systems, one brand.** Finding 3 is the root cause of finding 1. A single `--brand` custom property referenced from all three stylesheets is a ~10-line change that pays off every time the design changes.
- **Two icon systems**, finding 6, is fixable in the React components without touching the blob — the blob will keep using Bootstrap Icons until it's migrated. That's fine, but stop mixing them inside the same shadcn components.

## Open questions for the user

1. **Persian digits (finding 12).** Do you want prices, booking IDs, dates, and analytics numbers rendered with Persian digits (۱۲۳۴۵) via `toLocaleString('fa-IR')`? This is a product/tone call, not a bug.
2. **Bootstrap Icons fallback (finding 13).** Are you OK adding an `@fontsource-*`-style local dependency to remove the cdnjs runtime dependency, or should we leave the CDN in place and just add a graceful fallback?
3. **`window.confirm` → `AlertDialog` (finding 7).** Confirm you want all three admin destructive actions moved to shadcn `AlertDialog` (matches `my-bookings.tsx` already).
4. **Scope of duplicate `<main>` fix (finding 4).** Preference: remove the outer React `<main>` wrapper (cleaner, blob keeps its ID for anchor links) or strip the blob's `<main>` and keep the React one (more idiomatic for a React app but a bigger blob edit)?
5. **Icon swap (finding 6).** OK to introduce ~5–8 additional `lucide-react` icons (Gamepad2, Zap, Tag, HelpCircle, CalendarPlus, Phone) to replace `bi-*` in React components? No new dependency — lucide is already installed.
6. **Axe-core in CI (finding 14).** Worth adding a Playwright + axe-core smoke test to catch future blob a11y regressions, or out of scope?
